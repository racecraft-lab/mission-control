# Screenshot Guide

README screenshots are generated programmatically from Mission Control via Playwright.
The single source of truth is [`docs/screenshot-manifest.json`](./screenshot-manifest.json) â€”
each panel entry maps a PNG file to a URL plus the source globs that govern drift detection.

## Files in this directory

| File | Generated by | Purpose |
|------|--------------|---------|
| `mission-control-*.png` | `pnpm screenshots:refresh` | README screenshots |
| `screenshot-manifest.json` | hand-edited | panel â†’ PNG â†’ URL â†’ source globs |
| `_captures/` | `pnpm screenshots:capture --baseline` | CI capture output (gitignored) |
| `_diffs/` | `pnpm screenshots:diff` | pixelmatch overlays (gitignored) |

## How drift detection works

Two GitHub Actions workflows enforce screenshot freshness:

### Tier 1 â€” `screenshot-drift.yml` (always runs on visual-file PRs)
- Triggered when any file under `src/components/**`, `src/app/**/page.tsx`, etc. changes.
- Reads `screenshot-manifest.json`. For each panel, glob-matches its `sources` against the PR's changed files.
- Posts (or updates) one `<!-- screenshot-drift -->` PR comment naming exactly the PNGs whose source files were touched.
- Adds the `screenshot-drift` label.
- Add the **`skip-drift`** label to opt out of the comment for non-visual changes.

### Tier 2 â€” `screenshot-visual-diff.yml` (opt-in)
- Triggered when the PR carries the **`run-visual-diff`** label.
- Boots `pnpm start` against a fresh seeded SQLite DB inside the runner.
- Calls `pnpm screenshots:capture --baseline` â†’ writes `docs/_captures/*.png`.
- Calls `pnpm screenshots:diff` â†’ pixelmatches each capture against `docs/*.png`.
- Posts a per-panel pass/fail table and uploads diffs as a CI artifact.
- **Fails the PR** when any panel exceeds `manifest.diff.thresholdPixels`.

## Refreshing screenshots locally

### One-time auth setup (against a remote deployment)

```bash
# Pops a Chrome window â€” log in, then close. Saves cookies/localStorage to .mc-auth.json (gitignored).
MC_URL=https://hal.boreal-fir.ts.net:3000 pnpm screenshots:auth
```

### Capture all panels

```bash
MC_URL=https://hal.boreal-fir.ts.net:3000 \
MC_STORAGE_STATE=.mc-auth.json \
pnpm screenshots:refresh
```

### Capture a single panel

```bash
MC_URL=https://hal.boreal-fir.ts.net:3000 \
MC_STORAGE_STATE=.mc-auth.json \
pnpm screenshots:refresh --only=security
```

### Capture against local dev

```bash
# In one shell:
pnpm dev

# In another:
AUTH_USER=admin AUTH_PASS='your-local-pass' \
MC_URL=http://127.0.0.1:3000 \
pnpm screenshots:refresh
```

### Compare local captures against committed baselines

```bash
pnpm screenshots:capture --baseline   # writes docs/_captures/*.png
pnpm screenshots:diff                 # writes docs/_diffs/*.png + summary.json
```

## Updating the manifest

When you add or remove a panel, edit `screenshot-manifest.json`:

```json
{
  "id": "new-panel",
  "png": "docs/mission-control-new-panel.png",
  "url": "/new-panel",
  "sources": [
    "src/components/panels/new-panel.tsx"
  ]
}
```

Optional fields:
- `postNavigateAction: "click-tab:Graph"` â€” clicks a tab/button by visible text after navigation. Used by the `memory-graph` panel to switch into the graph view of the memory page.

## Optimising PNGs

The capture script writes raw `fullPage` PNGs from headless Chromium at 1440Ă—900 with `deviceScaleFactor=2`. To shrink before committing:

```bash
brew install pngquant   # macOS
pngquant --quality=80-95 --force --ext .png docs/mission-control-*.png
```

`pixelmatch` tolerates lossy compression as long as both sides use the same scheme, so optimised PNGs stay valid baselines.

## Banner suppression

Live deployments show transient banners (OpenClaw update, doctor warnings, the live-feed side panel). The capture script hides them via:
1. `manifest.navigation.hideSelectors` â€” CSS class globs.
2. A textContent-based heuristic that hides any near-top div mentioning "OpenClaw update / state integrity / doctor".

If your environment surfaces a banner the capture doesn't catch, add a selector to `manifest.navigation.hideSelectors`.
